Project: ~/Projects/kernel/

Plan: Two Bugs + One Enhancement
Bug 1: File slot exhaustion after ~8 commands
Root cause: fs_open() in fs/fs.c:91-108 never checks if an ext2 file is already open. Each call allocates a fresh file_t slot. The shell's pattern — fd_open() to check existence, then execve → process_exec → fs_open() again — burns 2 slots per command. With FS_MAX_FILES=16 (1 initrd test.txt + 15 ext2), slots exhaust after 7-8 commands.
Fix: After ext2_resolve succeeds, search all files[] for a matching ext2_ino before creating a new entry. Same file → same slot.
Bug 2: pwd returns blank (or /) after cd /subdir
Root cause: SYSCALL_GETCWD in kernel/kernel.c:355 calls ext2_find_name() to resolve each inode in the chain back to a name. If this fails (e.g. directory entry anomaly), err is set and the syscall returns -1. The CMD_PWD user code only prints if return >= 0, so the user sees nothing (or just \n).
Fix: When ext2_find_name fails, generate a numeric fallback "inode_N" instead of aborting the whole chain. pwd always produces output.
Enhancement: cd error reporting
Root cause: user/shell.c:97-99 ignores the return value of SYSCALL_CHDIR. Silent failures are confusing.
Fix: Check return value and print "cd: <path>: no such directory\n" on failure.
